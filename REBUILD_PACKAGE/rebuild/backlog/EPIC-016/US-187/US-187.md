# US-187: Autorisatie voor Beheer (beheerderrol)

## Doel
Toegang tot Beheer‑tab en configuratiewijzigingen beperken tot geautoriseerde beheerders; audit alle beheeracties.

## Acceptatiecriteria
- [ ] Beheer‑tab alleen zichtbaar voor beheerderrol (feature‑flag tijdelijk toegestaan)
- [ ] Opslaan van wijzigingen alleen toegestaan voor beheerderrol
- [ ] Audit van alle beheeracties met user/tijd/actie/details
- [ ] Voor nu: simpele rol via env/flag; later integratie met echte auth

## Definition of Done
- [ ] Feature‑flag/rol‑check geïmplementeerd
- [ ] UI‑knoppen disabled/hidden voor niet‑beheerders
- [ ] Audit aanwezig
- [ ] Documentatie bijgewerkt

## Rollen & autorisatie
- Roles: admin (beheerder), user (standaard). In deze US: simpele env/flag‑gebaseerde rol (bijv. FEATURE_ADMIN=true).
- Beveiliging UI: Beheer‑tab verborgen/disabled voor niet‑admins.
- Beveiliging service: server‑zijde controle bij POST/PUT op config‑endpoints.

## Audit
- Elke beheeractie (create/update/delete config) loggen met user + details.

## Toekomstig
- Integratie met echte auth (OIDC/SAML) en role‑based access control.

## Testplan
- Unit: rol‑check helpers; flags; server‑side guard
- Integratie: UI verbergt/disablt mutatie‑acties zonder rol; audit voor beheeracties
