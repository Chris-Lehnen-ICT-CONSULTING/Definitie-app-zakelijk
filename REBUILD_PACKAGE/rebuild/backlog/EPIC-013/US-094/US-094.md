# US-094: Beveiliging & privacy – portal output

## Gebruikersverhaal
**Als** security officer wil ik borgen dat de portal geen gevoelige data prijsgeeft, **zodat** we aan BIO/AVG voldoen.

## Acceptatiecriteria
- [ ] Geen secrets/PII in portal-index.json of UI
- [ ] Respecteer `canonical: false`/`status: archived` (optionele hide‑filter)
- [ ] Basis toegankelijkheid (ARIA labels, semantische HTML)

## Implementatie‑notities
- Filter/sanitize velden; optionele exclude‑patterns in config
- Snelle WCAG‑checks (kleurcontrast, focus)

## Verificatie
- Handmatige review + grep op gevoelige velden
- A11y‑check met Lighthouse/axe (basis)
