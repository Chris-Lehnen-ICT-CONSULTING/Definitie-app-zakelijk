# US-016: SRU Legal Database Integratie

## Gebruikersverhaal
**Als** officier van justitie (OM)  
**wil ik** toegang tot officiële juridische databases  
**zodat** definities gezaghebbende juridische bronnen refereren

## Acceptatiecriteria
- [ ] SRU‑zoekopdracht (KB + Rechtspraak) met CQL  
- [ ] Parsing van Dublin Core/XML naar kernvelden  
- [ ] Relevantie‑filtering en bronvermelding
- [ ] Timeouts (≤ 5s) en caching (TTL configurable)

### Niet‑functionele acceptatiecriteria (Veilige Scraping)
- Robots.txt/TOS respecteren; timeout ≤ 5s; rate limiting + backoff  
- Markup sanitization; geen PII; AVG/BIR‑conform  
- Eenduidige User‑Agent; volledige provenance; beknopte audit logging

## Definition of Done
- [ ] Unit tests (happy/timeout/XML edge cases)
- [ ] Integratietest met default config (mock)
- [ ] Geen broken links in Portal en provenance klikbaar
