# US-070: UI‑facade consolidatie (DefinitionUIService) + splitsing tabbed_interface.py

## Gebruikersverhaal
Als frontend engineer wil ik dat UI‑componenten via één facade service werken en grote modules opgesplitst zijn, zodat de UI eenvoudiger te testen en onderhouden is.

## Acceptatiecriteria (SMART)
- Specifiek: alle repository/workflow calls via `DefinitionUIService`; `tabbed_interface.py` opgesplitst in logische subcomponenten (< 500 regels/module).
- Meetbaar: 0 directe repo‑calls in UI; tests voor facade‑paden aanwezig.
- Tijdgebonden: sprint‑38.

## Testplan
- tests/ui/test_ui_facade_usage.py – monkeypatch en assert dat facade wordt gebruikt.

## Definition of Done
- UI blijft functioneel; code‑grootte teruggebracht; componentgrenzen gedocumenteerd.
