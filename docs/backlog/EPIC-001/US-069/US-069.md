# US-069: Prompt Builder Compaction Logic

## Gebruikersverhaal
Als platform engineer wil ik dat de prompt builder automatisch compacte promoten construeert binnen een tokenbudget zodat we consistent onder de model-limieten blijven zonder relevante context te verliezen.

## Context en Herkomst
- TODO in code: `src/services/prompts/modular_prompt_builder.py.backup:884` – “Implementeer compactie logica indien gewenst”.
- REQ-018 (Core Definition Generation) benoemt een modulaire PromptServiceV2.

## Acceptatiecriteria
- [ ] Tokenbudget afdwinging: max X tokens (configureerbaar) per promptdeel en totaal.
- [ ] Deduplcatie: identieke/sterk overlappende contextregels worden samengevoegd of verwijderd.
- [ ] Prioritering: juridisch/wettelijk > organisatorisch > historisch/technisch bij overschrijding.
- [ ] Samenvatten: lange contextitems worden automatisch samengevat (doel: 30–50% reductie) met behoud van kernbegrippen.
- [ ] Meetbaar: rapportage van reductiepercentage en gebruikte tokens in metadata.
- [ ] Deterministisch bij gelijke input (zelfde output bij gelijke contextset).

## Technische Uitwerking
### Componenten
- `PromptCompactor` (onderdeel van PromptServiceV2):
  - `compact_sections(sections: list[PromptSection], budget: TokenBudget) -> list[PromptSection]`
  - Strategieën: deduplication, truncation per sectie, extractive summarization voor overschot.

### Configuratie
```yaml
prompt:
  token_budget_total: 2800
  token_budget_sections:
    juridisch: 1200
    wettelijk: 900
    organisatorisch: 400
    overige: 300
  summarization_ratio: 0.6  # reduce to 60% van oorspronkelijke lengte
```

### Telemetrie
- Schrijf `prompt_compaction` metadata: `original_tokens`, `final_tokens`, `reduction_pct`, `dropped_items`.

## Tests
- Unit: deduplicatie, prioritering bij budgetoverschrijding, samenvatten > reductiedoel, determinisme.
- Integratie: volledige promptconstructie blijft < tokenbudget en behoudt kerninformatie (assert op aanwezigheid kerntermen).
