# US-055: External Sources Governance Policy

## Gebruikersverhaal

**Als een** security officer
**Wil ik** een duidelijke governance policy voor externe databronnen
**Zodat** we compliant zijn met AVG, BIO en justice sector beveiligingsstandaarden

### Compliance Frameworks

- **AVG/GDPR (Verordening 2016/679)**
  - Artikel 5: Beginselen (rechtmatigheid, doelbinding, minimale gegevensverwerking)

  - Artikel 32: Beveiliging van verwerking
  - Artikel 35: DPIA vereisten
  - Artikel 37-39: Functionaris Gegevensbescherming rol

- **BIO 2019 (Baseline Informatiebeveiliging Overheid)**
  - Strategisch: Risicomanagement framework
  - Tactisch: Beveiliging governance processen
  - Operationeel: 113 security controls
  - BIO-thema 12.1: Operationele procedures
  - BIO-thema 14.1: Beveiliging in relaties met leveranciers

- **NEN-ISO/IEC 27001:2017 & 27002:2022**
  - ISMS vereistes
  - Control objectives A.5-A.18
  - Supply chain security (A.15)
  - Information transfer (A.13)

#### Justice Sector Specifiek
- **Wet justitiële en strafvorderlijke gegevens (Wjsg)**

  - Artikel 51a: Bewaartermijnen

- **Aanwijzing gegevensverstrekking OM**
  - 2021A003: Kaders voor datadeling
  - Privacy protocollen per ketenpartner
  - Convenant gegevensuitwisseling strafrechtketen

- **DJI Beveiligingsvoorschrift**
  - Classificatie: Openbaar, Intern, Vertrouwelijk, Geheim
  - Zonering: Beveiliging zones A-D
  - Incident response procedures

- **Rechtspraak Open Data Beleid**
  - ECLI standaard voor uitspraken
  - Anonimisering vereisten
  - Metadata standaarden

### Policy Onderdelen

#### 1. Data Classificatie Framework
```
┌───────────────────┬──────────────────┬───────────────────┐
│ Classificatie    │ Voorbeelden      │ Beveiliging Controls │
├───────────────────┼──────────────────┼───────────────────┤
│ Openbaar        │ Uitspraken      │ TLS 1.2+         │
│                 │ Wet teksten     │ Integrity checks  │
├───────────────────┼──────────────────┼───────────────────┤
│ Intern          │ Werkprocessen   │ + Access control  │
│                 │ Richtlijnen     │ + Audit logging   │
├───────────────────┼──────────────────┼───────────────────┤
│ Vertrouwelijk   │ Dossiers       │ + Encryption      │
│                 │ Persoonsdata    │ + DLP            │
├───────────────────┼──────────────────┼───────────────────┤
│ Geheim          │ Staatsgeheim    │ + Air gap        │
│                 │ Onderzoeksdata  │ + HSM            │
└───────────────────┴──────────────────┴───────────────────┘
```

#### 2. Approved Sources Registry
**Whitelist (Goedgekeurd)**
- Rechtspraak.nl (openbare uitspraken)
- EUR-Lex (EU wetgeving)
- Wetten.nl/BWB (Nederlandse wetgeving)
- Wikipedia (algemene context)
- Officiële overheids APIs

**Greylist (Conditioneel)**
- Academic databases (met licentie)
- News sources (met source attribution)
- Commercial legal databases (met DPA)

**Blacklist (Verboden)**
- Social media platforms
- Ongevalideerde user content
- Tor/dark web sources
- Non-EU data brokers

#### 3. Risk Assessment Framework
**Data Source Risk Matrix**
- Jurisdiction (EU/non-EU)
- Data controller status
- Beveiliging maturity
- Incident history
- Compliance certifications

**Assessment Templates**
- Quick scan (<1 dag)
- Standard assessment (1 week)
- Full assessment (2-4 weeks)

#### 4. Privacy Impact Assessment (PIA/DPIA)
**Trigger Criteria**
- Nieuwe externe databron
- Grootschalige verwerking
- Bijzondere persoonsgegevens
- Systematische monitoring
- Data matching/profiling

**PIA Process**
1. Necessity & proportionality test
2. Risk identification
3. Mitigation measures
4. Residual risk acceptance
5. FG consultatie

#### 5. Incident Response Protocol
**Incident Types**
- Data breach (meldplicht <72u)
- Unauthorized access
- Data corruption
- Availability loss
- Compliance violation

**Response Team**
- Incident Commander (CISO)
- Legal Officer (Privacy)
- Technical Lead
- Communication Officer
- External Provider Liaison

#### 6. Vendor Management
**Due Diligence Criteria**
- ISO 27001 certificering
- SOC 2 Type II rapport
- ISAE 3402 verklaring
- Sub-processor register
- Exit strategie

### Key Policy Elements

#### 1. Source Approval Governance
**Approval Authority Matrix**
| Data Type | Approval Level | SLA |
|-----------|---------------|-----|
| Openbaar | Team Lead | 1 dag |
| Intern | Manager | 3 dagen |
| Vertrouwelijk | Director | 1 week |
| Geheim | Board | 2 weken |

#### 2. Data Retention & Deletion
**Retention Schedule**
- Definitie output: 5 jaar (wettelijk)
- Validatie logs: 1 jaar (audit)
- API responses: 30 dagen (cache)
- Personal data: Minimaal noodzakelijk
- System logs: 90 dagen (security)

**Deletion Procedures**
- Automated purging via retention policies
- Secure wipe (3x overwrite)
- Certificate of destruction
- Backup inclusion verificatie

#### 3. Cross-Border Transfer Controls
**EU/EEA Transfers**
- Geen additionele vereisten
- Standard contractual clauses (SCC)

**Third Country Transfers**
- Adequacy decision check
- Transfer impact assessment (TIA)
- Supplementary measures
- Binding corporate rules (BCR)

#### 4. Encryption Standards
**Data at Rest**
- AES-256-GCM minimum
- Key management via HSM
- Database transparent encryption
- File system encryption

**Data in Transit**
- TLS 1.3 preferred, 1.2 minimum
- Certificate pinning voor kritieke APIs
- VPN voor management traffic
- End-to-end encryption voor Geheim

#### 5. Access Control Framework
**RBAC Model**
```yaml
roles:
  viewer:
    - read:public_sources
    - read:definitions
  analyst:
    - inherits: viewer
    - read:internal_sources
    - write:definitions
  admin:
    - inherits: analyst
    - manage:sources
    - audit:access_logs
  security_officer:
    - audit:all
    - manage:compliance
```
