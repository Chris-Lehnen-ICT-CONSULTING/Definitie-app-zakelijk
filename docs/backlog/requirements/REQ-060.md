# Requirement REQ-060: Health Check Endpoints

## Beschrijving
Het systeem moet health check endpoints bieden voor monitoring tools OM de status van de applicatie en zijn Afhankelijkheden te controleren.

## Rationale
Health checks zijn essentieel voor automated monitoring, load balancer configuratie, en kubernetes liveness/readiness probes.

## Acceptatiecriteria (SMART)
- /health endpoint voor basic liveness
- /ready endpoint voor readiness check
- Dependency checks (database, API keys)
- Response time < 1 seconde
- Structured JSON response

## Uitwerking / Notities
Health check responses:
- Status: healthy/unhealthy
- Versie information
- Dependency status
- Response time metrics
- Last check timestamp
