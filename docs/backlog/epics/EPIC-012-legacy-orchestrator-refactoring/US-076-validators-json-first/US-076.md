# US-076: Validators JSON-first met gedeelde basisklasse

## Story Overview
**ID**: US-076
**Epic**: EPIC-012 - Legacy Orchestrator Refactoring
**Status**: TODO
**Priority**: MEDIUM
**Points**: 8

## User Story
Als kwaliteitsengineer wil ik dat toetsregel-validators primair JSON-gedreven zijn met een gedeelde basisklasse, zodat we minder boilerplate-validators hebben en consistente scoring/berichten.

## Business Context
Elke regel heeft JSON + vaak een Python-validator met vergelijkbare structuur. Een generiek pad (basisklasse) dat JSON-config leest kan >80% van de regels dekken; maatwerk alleen waar nodig.

## Acceptance Criteria
- [ ] Basisklasse `BaseRuleValidator` met hook-points (patterns, structure, examples)
- [ ] Generieke factory `create_validator_from_json(regel_id)`
- [ ] 80%+ regels zonder maatwerk-Python
- [ ] Consistente returnvorm (success, message, score) conform V2 mapping

## Technical Implementation
- Ontwerp `BaseRuleValidator` in `toetsregels/validators/base.py` met:
  - `validate(definitie, begrip, context) -> (bool, str, float)`
  - helpers voor patterns/voorbeelden/structuur
- Schrijf adapter die JSON inleest via `manager.load_regel(regel_id)` en instanties maakt.
- Migreer bestaande per-regel-Python waar mogelijk naar generiek; laat uitzonderingen bestaan.

## Testing Requirements
- Unit tests voor basisklasse en generieke mapping.
- Golden tests per categorie (STR/INT/SAM/ESS/VER/CON/ARAI) met voorbeeldcases.

## Migration Plan
1. Basisklasse + factory.
2. Gefaseerde migratie per categorie.
3. Verwijder onnodige per-regel-Python.

## Dependencies
- `toetsregels/manager.py`, `toetsregels/validators/*`, `validation` pipeline V2.

## Definition of Done
- [ ] JSON-first validators actief
- [ ] 80%+ regels generiek
- [ ] Tests en documentatie compleet
