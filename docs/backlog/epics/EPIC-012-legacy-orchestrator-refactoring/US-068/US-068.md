# US-068: Verwijder Domein Restanten uit Codebase

## User Story
**Als** ontwikkelaar
**Wil ik** alle restanten van het deprecated 'domein' veld verwijderen
**Zodat** de codebase consistent is met EPIC-010 context model

## Context
EPIC-010 heeft 'domein' vervangen door expliciete context velden (organisatorisch/juridisch/wettelijk).
Er zijn nog restanten in verschillende modules die verwarring kunnen veroorzaken.

## Gevonden Restanten
1. `src/services/definition_generator_monitoring.py` - Metrics gebruikt `definition.domein`
2. `src/services/export_service.py` - Export bevat 'domein' in metadata
3. `src/services/definition_generator_context.py` - Constants/comments voor domein
4. `src/services/definition_generator_enhancement.py` - integrate_domain functie
5. `src/services/modern_web_lookup_service.py` - Domein module availability check
6. Database schema/migrations - Legacy domein kolom

## Acceptatiecriteria
- [ ] Geen 'domein' referenties meer in actieve code
- [ ] Export/monitoring gebruikt alleen moderne context velden
- [ ] Database migratie voor domein kolom removal
- [ ] Alle tests blijven groen
- [ ] Geen functionele impact op V2 flow
