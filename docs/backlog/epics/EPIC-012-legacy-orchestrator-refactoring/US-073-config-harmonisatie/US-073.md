# US-073: Config-harmonisatie en verwijderen dotenv uit productiepad

## Story Overview
**ID**: US-073
**Epic**: EPIC-012 - Legacy Orchestrator Refactoring
**Status**: TODO
**Priority**: HIGH
**Points**: 3

## User Story
Als devops engineer wil ik één consistente configuratiebron zodat gedrag tussen omgevingen voorspelbaar is en secrets veilig via env/CI worden beheerd.

## Business Context
`src/services/ai_service.py` laadt `.env` (dotenv) terwijl `src/main.py` expliciet stelt geen `.env` te gebruiken. `config_manager` biedt al YAML + env overrides. Harmonisatie voorkomt verrassingen in productie.

## Acceptance Criteria
- [ ] Geen directe `load_dotenv()` in productiecode
- [ ] Alle defaultwaarden komen uit `config_manager` YAML + env overrides
- [ ] Documentatie van vereiste env-vars/CI-secrets is compleet
- [ ] Faal vroeg wanneer vereiste secrets ontbreken buiten dev

## Technical Implementation
- Verwijder/conditioneer `load_dotenv()` uit `services/ai_service.py` (alleen dev scripts).
- Borg dat `get_default_model/temperature` enz. uit `config_manager` komen.
- Voeg “config validation” stap toe tijdens app-start (warning→error in non-dev).

## Testing Requirements
- Unit: config-manager leest YAML en env overrides correct.
- Integratie: AI-service gebruikt config-manager defaults, geen .env afhankelijkheid.

## Migration Plan
1. Verplaatsen dotenv-usage naar dev-only scripts.
2. Update docs/READMEs en CI pipeline secrets.
3. Tests en runbooks bijwerken.

## Dependencies
- `config/config_*.yaml`, `config_manager`, AI-services.

## Definition of Done
- [ ] Geen dotenv in core services
- [ ] Config gedocumenteerd en gevalideerd
- [ ] CI secrets ingesteld
