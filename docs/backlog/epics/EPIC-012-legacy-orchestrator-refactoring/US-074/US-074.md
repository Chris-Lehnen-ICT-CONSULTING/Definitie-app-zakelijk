# US-074: Import-hygiëne en verwijderen sys.path-hacks

## Story Overview
**ID**: US-074
**Epic**: EPIC-012 - Legacy Orchestrator Refactoring
**Status**: TODO
**Priority**: HIGH
**Points**: 5

## User Story
Als developer wil ik betrouwbare imports zonder `sys.path`-manipulaties zodat modules correct via package-namespaces laden en type-checking/IDE’s goed werken.

## Business Context
Diverse modules voegen paden toe aan `sys.path` (UI-componenten, tools, entrypoints). Dit fragiliseert imports en belemmert packaging/CI.

## Acceptance Criteria
- [ ] Geen `sys.path.append/insert` in app-code buiten strikt noodzakelijke entrypoints/tools
- [ ] `external/`, `orchestration/`, `analysis/`, `monitoring/` etc. zijn volwaardige packages met `__init__.py` en absolute imports
- [ ] UI `ExternalSourcesTab` gebruikt `from external.external_source_adapter import ...`

## Technical Implementation
- Voeg ontbrekende `__init__.py` toe waar nodig.
- Vervang relatieve hack-imports door absolute package imports.
- Laat tooling/scripts (in `scripts/`) een eigen PYTHONPATH gebruiken indien nodig, niet app-code.

## Testing Requirements
- Static tests (CI) die `sys.path`-wijzigingen in src/ blokkeren.
- Runtime rooktests (import tree) slagen zonder hacks.

## Migration Plan
1. Inventariseren hacks (rg-lijst gereed).
2. Packages corrigeren en imports updaten.
3. CI guard toevoegen.

## Dependencies
- UI componenten, external adapters, monitoring/analysis modules.

## Definition of Done
- [ ] Geen path-hacks
- [ ] Alle imports stabiel
- [ ] CI guard actief
