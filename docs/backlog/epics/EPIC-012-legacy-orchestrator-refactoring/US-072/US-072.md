# US-072: Export-consolidatie en opschoning export-mappen

## Story Overview
**ID**: US-072
**Epic**: EPIC-012 - Legacy Orchestrator Refactoring
**Status**: TODO
**Priority**: MEDIUM
**Points**: 3

## User Story
Als gebruiker wil ik betrouwbare export van definities zodat ik consistente TXT/CSV/PDF-exporten heb via één servicepad.

## Business Context
Er zijn twee exportpaden zichtbaar: `src/export/` (actief via `services/export_service.py`) en een lege/legacy `src/exports/`. Dit veroorzaakt verwarring en risico op verkeerde imports.

## Acceptance Criteria
- [ ] Eén exportpad (`src/export/`) is authoritative
- [ ] `src/exports/` verwijderd of gearchiveerd (geen codeverwijzingen)
- [ ] `ExportService` biedt eenduidige API voor UI en services
- [ ] Tests voor exportservice (TXT/CSV) zijn groen

## Technical Implementation
- Bevestig dat `src/exports/` nergens gebruikt wordt (rg/CI).
- Verwijder map of archiveer met duidelijke README-verwijzing.
- Borg dat `services/export_service.py` de enige ingress is voor export.

## Testing Requirements
- Unit tests: TXT/CSV export pad, foutafhandeling, juiste encodings.
- Integratie: UI exportknop routeert via `ExportService`.

## Migration Plan
1. Verwijzingen verifiëren (CI/grep).
2. Map `src/exports/` verwijderen en imports fixen indien nodig.
3. Tests uitvoeren en documentatie bijwerken.

## Dependencies
- UI exportknoppen in `ui/components_*` en `ui/services/definition_ui_service.py`.

## Definition of Done
- [ ] Eén exportpad
- [ ] Tests groen
- [ ] Documentatie aangepast
