# US-184: Contextopties beheren (organisaties/rechtsgebieden/wetgeving)

## Doel
Beheer van keuzelijsten voor de Context Selector: Organisatorische contexten, Juridische contexten en Wettelijke basis.

## Acceptatiecriteria
- [ ] CRUD op drie lijsten (organisatorisch/juridisch/wetgeving) via UI
- [ ] Validatie op lengte/tekens; dubbele items voorkomen
- [ ] Opslag in DB (config) met audit trail
- [ ] Direct zichtbaar in Context Selector (hot‑reload)
- [ ] Export/Import (JSON/YAML)

## Definition of Done
- [ ] Beheer UI per lijst
- [ ] DB‑opslag + audit trail
- [ ] ContextManager leest actuele lijsten
- [ ] Documentatie bijgewerkt

## Data‑model
- Tabel `context_options` (id, group, value, sort_order, active, created_at, updated_at)
- group ∈ {org, jur, wet}

## UI‑Details
- CRUD per groep; sorteerbaar; deactiveer ipv verwijderen.
- Validatie: unieke waarde binnen groep (case‑insensitive), max lengte.

## Integratie
- EnhancedContextManagerSelector leest opties uit DB via ContextManager.
- Hot‑reload bij wijziging.

## Import/Export
- JSON/YAML lijst per groep.

## Testplan
- Unit: CRUD op context_options; unieke waarden per groep; sortering; active toggle
- Integratie: wijzig lijst → UI selector toont update zonder restart; ContextManager leest nieuwe set
- UI: validaties, duplicate preventie, audit entry
