# US-107: Portal Tabs – Architectuur (EA/SA/TA) en Database Structuur

## Gebruikersverhaal
Als gebruiker van de documentatie‑portal wil ik aparte tabbladen voor Architectuur en Database Structuur zodat ik snel kan navigeren naar EA/SA/TA‑documenten en het databaseschema, en deze oplossing herbruikbaar is voor andere projecten.

## Context
- Portal is vastgelegd onder [EPIC‑013: Centrale Documentatie‑Portal](../EPIC-013.md).
- Dit is een uitbreiding met configureerbare tabs zodat de portal generiek inzetbaar blijft.

## Doelen
- Tab “Architectuur” met snelle toegang tot EA/SA/TA.
- Tab “Database Structuur” met schema‑overzicht en richtlijnen.
- Configurabel (padmapping) zodat portaal ook in andere repos werkt.

## Scope
In scope
- Portal UI: twee nieuwe tabs (Architectuur, Database).
- Datamapping: bronbestanden voor beide tabs + fallback.
- Configuratie: JSON/YAML in `docs/portal/` voor padmapping.

Out of scope
- Diepe inhoudswijzigingen in de genoemde documenten (alleen navigatie/portaal UI).

## Bronnen (links gecontroleerd)
- Architectuur (EA/SA/TA)
  - EA: `docs/architectuur/ENTERPRISE_ARCHITECTURE.md`
  - SA: `docs/architectuur/SOLUTION_ARCHITECTURE.md`
  - TA: `docs/architectuur/TECHNICAL_ARCHITECTURE.md`
- Database Structuur
  - Schema: `src/database/schema.sql`
  - Richtlijnen: `docs/guidelines/DATABASE_GUIDELINES.md`
  - Migratie: `src/database/migrate_database.py`

## UI‑Specificatie (MVP)
- Tabs in portal (bovenin): [Backlog] [Guidelines] [Architectuur] [Database] …
- Architectuur tab: drie kaarten/links (EA/SA/TA) + korte toelichting en last_verified.
- Database tab: links naar schema.sql, migratie, DB‑richtlijnen; extra kaart met “Active DB: data/definities.db”.

## Configuratie (hergebruik in andere projecten)
`docs/portal/config.json` (voorbeeld)
```json
{
  "tabs": [
    { "id": "architecture", "label": "Architectuur", "links": [
      { "label": "EA", "path": "docs/architectuur/ENTERPRISE_ARCHITECTURE.md" },
      { "label": "SA", "path": "docs/architectuur/SOLUTION_ARCHITECTURE.md" },
      { "label": "TA", "path": "docs/architectuur/TECHNICAL_ARCHITECTURE.md" }
    ]},
    { "id": "database", "label": "Database", "links": [
      { "label": "Schema", "path": "src/database/schema.sql" },
      { "label": "Migratie", "path": "src/database/migrate_database.py" },
      { "label": "Richtlijnen", "path": "docs/guidelines/DATABASE_GUIDELINES.md" }
    ]}
  ]
}
```

## Acceptatiecriteria
- [ ] Portal toont tab “Architectuur” met drie links (EA/SA/TA) die openen/naar bron navigeren.
- [ ] Portal toont tab “Database Structuur” met schema/migratie/richtlijnen.
- [ ] Tabs zijn config‑gedreven (aan/uit en padmapping via `docs/portal/config.json`).
- [ ] Links in deze US verwijzen naar bestaande paden (gecontroleerd).

## Teststrategie (concept)
- Handmatig: open `docs/portal/index.html`, controleer tabs/links.
- Config‑toggle: verwijder/voeg een tab in config.json, portal past zich zonder codewijziging aan.

## Risico’s
- Verplaatsing van documenten kan links breken — mitigatie: portal generator gebruikt frontmatter/traceability voor verificatie.
