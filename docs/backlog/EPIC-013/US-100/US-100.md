# US-100: Portal UI – tabs/views (Planning, Requirements, Alles)

## Gebruikersverhaal
**Als** gebruiker wil ik snel kunnen wisselen tussen views (Alles/Planning/Requirements), **zodat** ik gericht kan werken zonder telkens filters opnieuw te zetten.

## Acceptatiecriteria
- [ ] Tabs/anchors: Alles, Planning, Requirements
- [ ] Per view eigen defaults (sorteren/filters); terug naar “Alles” is één klik
- [ ] View staat in URL fragment (deeplink en bookmarkbaar)

## Implementatienotities
- Kleine router op hashchange (client‑side)
- Bewaar gekozen filters in sessionStorage per view

## Verificatie
- Directe link naar `#view=planning` opent planningweergave
- Wisselen van tabs verandert filters/sort en is bookmarkbaar
