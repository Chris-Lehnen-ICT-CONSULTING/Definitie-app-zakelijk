# US-100: Portal UI – tabs/views (Planning, Requirements, Alles)

## Gebruikersverhaal
**Als** gebruiker wil ik snel kunnen wisselen tussen views (Alles/Planning/Requirements), **zodat** ik gericht kan werken zonder telkens filters opnieuw te zetten.

## Acceptatiecriteria
- [x] Tabs/anchors: Alles, Planning, Requirements (+ Req Matrix, Mijn Werk)
- [x] View staat in URL fragment (deeplink en bookmarkbaar)
- [x] Basis A11y voor tabs (role/aria)
- [ ] View-specifieke defaults (sort/filters) bewaren (vervolgstap)

## Implementatienotities
- Hashchange-router actief (client‑side); query `#q=…` gesynchroniseerd
- Filters (owner/status/prio/alleen-US) in “Mijn Werk” gesynchroniseerd naar hash
- Bewaren van view-specifieke defaults (sessionStorage) valt buiten MVP

## Verificatie
- Directe link naar `#view=planning` opent planningweergave
- Wisselen van tabs is bookmarkbaar; A11y ondersteuning voor focus/aria‑selected
