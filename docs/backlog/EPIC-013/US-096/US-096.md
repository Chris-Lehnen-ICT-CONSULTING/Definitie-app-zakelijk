# US-096: Release‑snapshots & audit trail (optioneel)

## Gebruikersverhaal
**Als** auditor wil ik per release een bevroren snapshot van de portaldata, **zodat** ik wijzigingen door de tijd kan vergelijken.

## Acceptatiecriteria
- [ ] Optionele snapshotoutput: `docs/portal/snapshots/{release}.json`
- [ ] Release‑tag triggert snapshot in CI
- [ ] Snapshot link zichtbaar in portal

## Implementatie‑notities
- CI stap bij tag: exporteer actuele `portal-index.json` naar snapshots/
- Markeer snapshot in de UI als read‑only

## Verificatie
- Maak testtag; controleer snapshotbestand en link in UI
