# Requirement REQ-030: Rule Priority System

## Beschrijving
Het systeem moet een prioriteringssysteem implementeren voor validatieregels met drie niveaus (high/medium/low) om kritieke fouten te onderscheiden van suggesties voor verbetering.

## Rationale
Niet alle validatiefouten zijn even kritiek. Een prioriteringssysteem helpt gebruikers zich te focussen op de belangrijkste problemen eerst, wat de efficiëntie van het definitieproces verhoogt.

## Acceptatiecriteria (SMART)
- Drie priority levels zijn gedefinieerd: high, medium, low
- Elke validatieregel heeft een toegewezen priority level
- UI toont high priority issues bovenaan
- Filtering op priority level is mogelijk
- Priority wordt consistent toegepast over alle 45 regels

## Uitwerking / Notities
- High: Blokkerende fouten (juridisch onjuist)
- Medium: Belangrijke verbeterpunten (kwaliteit)
- Low: Suggesties (stijl en optimalisatie)
