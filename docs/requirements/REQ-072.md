# Requirement REQ-072: Test Data Management

## Beschrijving
Het systeem moet een gestructureerd test data management systeem hebben met fixtures, factories, en seeders voor consistent en reproduceerbaar testen.

## Rationale
Goede test data management voorkomt flaky tests en maakt tests maintainable. Het zorgt ook voor AVG compliance door geen productie data in tests te gebruiken.

## Acceptatiecriteria (SMART)
- Pytest fixtures voor common test data
- Factory pattern voor dynamic test data
- Separate test database
- Data anonymization voor productie samples
- Test data Versieing

## Uitwerking / Notities
Test data strategy:
- JSON fixtures voor static data
- Factory Boy voor dynamic objects
- Faker voor realistic test data
- Database seeders voor integration tests
- No PII in test data

## Conflicten (indien van toepassing)
Privacy vereisten verbieden gebruik van echte gebruikersdata.
