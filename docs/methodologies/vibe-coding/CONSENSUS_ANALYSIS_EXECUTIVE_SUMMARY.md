## üéì Prompt Engineering Best Practices (Perplexity + Context7)

### From Perplexity Research:

**Fundamental Paradigm Shift**:
> "Vibe coding transforms development from telling machines exactly what to do into guiding them with intent"

**Comparison with Traditional Development**:

| Traditional | Vibe Coding |
|-------------|-------------|
| Syntax-focused | Intent-focused |
| Debug via stack traces | Debug via prompt refinement |
| Heavy setup | Fast experimentation |
| Weeks to validate | Days to validate |

### From Context7 (DAIR-AI Guide):

**Template Structure Best Practices**:
1. **Instruction**: What to do
2. **Context**: Background information
3. **Input Data**: The actual data to process
4. **Output Format**: Expected structure

**Example from Context7**:
```
Instruction: Summarize the following text into three bullet points.
Context: The following is an excerpt from a news article about climate change.
Input Data: [Article Text]
Output Format: Bulleted list
```

### From Context7 (Anthropic Tutorial):

**Key Patterns**:
1. **PREFILL**: Guide Claude's response start
2. **Few-Shot Prompting**: Provide 2-3 examples
3. **XML Tags**: Structure input/output (`<context>`, `<response>`)
4. **Precognition**: Encourage step-by-step thinking
5. **Output Formatting**: Specify exact structure

**Example from Anthropic**:
```python
PROMPT = f"<email>{EMAIL}</email>. Make this email more {ADJECTIVE}."
PREFILL = f"<{ADJECTIVE}_email>"
```
