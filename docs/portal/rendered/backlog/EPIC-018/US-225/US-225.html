US-225: DOCX/PDF extractors beschikbaar en actief

Gebruikersverhaal

Als gebruiker wil ik dat geüploade DOCX/PDF correct worden uitgelezen, zodat de documentcontext bruikbaar is bij genereren.

Acceptatiecriteria

  • [ ] DOCX extractie via `python-docx` actief; geen fallbackmelding in resultaten
  • [ ] PDF extractie via `PyPDF2` actief; geen fallbackmelding in resultaten
  • [ ] “Totale tekst” in UI toont realistische lengte (> 1.000 chars voor grote documenten)
  • [ ] Unsupported: `.doc` (legacy) — UI/Docs melden beperking

Technisch

  • Dependencies vastleggen (requirements)
  • Extractor behoudt bestaande interfaces; geen wijziging aan DocumentProcessor‑API

Tests

  • Unit tests voor DOCX/PDF paden (guarded bij ontbrekende libs)
