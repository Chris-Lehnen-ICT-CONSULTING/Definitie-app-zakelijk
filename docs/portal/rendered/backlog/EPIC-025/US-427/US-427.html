US-427: Split Gigantic UI Components

User Story

Als developer

Wil ik maintainable code files (< 500 LOC)

Zodat ik code kan begrijpen, testen en safely wijzigen

Description

Split 5 gigantic files die 2-4.7x over 500-regel limiet gaan:

  1. `definition_generator_tab.py`: 2339 LOC ‚Üí 3x ~700 LOC
  2. `definitie_repository.py`: 1800 LOC ‚Üí 3x ~600 LOC
  3. `tabbed_interface.py`: 1733 LOC ‚Üí 5x ~350 LOC

(US-427 focust op top 3, rest in US-433)

Acceptance Criteria

  • [ ] `definition_generator_tab.py` split in 3 files (UI, logic, renderer)
  • [ ] `definitie_repository.py` split in 3 files (read, write, bulk)
  • [ ] `tabbed_interface.py` split in 5 tab modules
  • [ ] All new files < 800 LOC (target 500)
  • [ ] All tests still pass
  • [ ] No functionality lost

Tasks

File 1: definition_generator_tab.py (2339‚Üí~700 each)

  1. Extract to `definition_generator_ui.py` (UI rendering logic)
  2. Extract to `definition_generator_logic.py` (Business logic)
  3. Extract to `definition_result_renderer.py` (Result display)
  4. Update imports, test

File 2: definitie_repository.py (1800‚Üí~600 each)

  1. Extract to `definitie_repository_read.py` (Query methods)
  2. Extract to `definitie_repository_write.py` (Insert/Update)
  3. Extract to `definitie_repository_bulk.py` (Bulk operations)
  4. Update container.py, test

File 3: tabbed_interface.py (1733‚Üí~350 each)

  1. Split to 5 separate tab modules in `ui/tabs/`
  2. Keep orchestration in `tabbed_interface.py` (thin coordinator)
  3. Update imports, test

Validation

# No files > 500 LOC
find src -name "*.py" -exec wc -l {} \; | awk '$1 > 500'
# Expected: (empty) or only a few close to limit

# Tests still pass
pytest -q
# Expected: All green

# Check imports updated
grep "definition_generator_tab import" src/**/*.py
# Expected: Updated to new modules
