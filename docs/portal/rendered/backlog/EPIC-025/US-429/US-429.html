US-429: Implement CI Quality Gates

User Story

Als developer

Wil ik dat CI quality gates afdwingt

Zodat code quality standards niet omzeild kunnen worden

Acceptance Criteria

  • [ ] CI verifieert dat pre-commit hooks zijn uitgevoerd
  • [ ] preflight-checks.sh wordt gerund in GitHub Actions
  • [ ] Commits zonder pre-commit execution worden geblokkeerd
  • [ ] Branch name validation toegevoegd (feature/*, fix/*, etc)
  • [ ] Failing checks blokkeren merge

Tasks

  1. Add `.github/workflows/quality-gates.yml`
  2. Verify pre-commit ran (check for hook metadata)
  3. Run `~/.ai-agents/preflight-checks.sh`
  4. Validate branch naming convention
  5. Test with deliberate violations

Validation

gh workflow list | grep "quality-gates"
