US-005: Centralized AI Model Configuration

Gebruikersverhaal

As a developer

wil ik all AI model settings in one place

zodat there are no hardcoded defaults scattered in the code

Acceptatiecriteria

SMART Acceptatiecriteria

  • **Specifiek:** [Exact gedrag dat moet worden gerealiseerd]
  • **Meetbaar:**
  • - Response tijd: < 200ms voor UI acties
  • - Processing tijd: < 5 seconden voor generatie
  • - Success rate: > 95% voor validaties
  • **Acceptabel:** Haalbaar binnen huidige architectuur
  • **Relevant:** Direct gerelateerd aan gebruikersbehoefte
  • **Tijdgebonden:** Gerealiseerd binnen huidige sprint

Criterion 1: No Hardcoded Models

gegeven the codebase is scanned

wanneer looking for model references

dan no hardcoded model names like "gpt-4" are found in code

Criterion 2: Single Configuration Source

gegeven AI model configuration is needed

wanneer any service requests model settings

dan they are retrieved from the central configuration

Criterion 3: Easy Model UpDatums

gegeven a new AI model Versie is available

wanneer updating the configuration

dan all services use the new model without code Wijzigingen

Specific

De gebruiker moet exact [ACTIE] kunnen uitvoeren met [RESULTAAT]

Measurable

Response tijd < 2 seconden, Success rate > 99%, Gebruikerstevredenheid > 8/10

Technische Notities

  • All model configs in `config/ai/models.yaml`
  • No hardcoded "gpt-4" strings in source code
  • Model selection via configuration only
  • Supports model Versieing en fallbacks
