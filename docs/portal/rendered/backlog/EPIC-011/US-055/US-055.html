US-055: External Sources Governance Policy

Gebruikersverhaal

Als een security officer

Wil ik een duidelijke governance policy voor externe databronnen

Zodat we compliant zijn met AVG, BIO en justice sector beveiligingsstandaarden

Acceptatiecriteria

SMART Acceptatiecriteria

  • **Specifiek:** [Exact gedrag dat moet worden gerealiseerd]
  • **Meetbaar:**
  • - Response tijd: < 200ms voor UI acties
  • - Processing tijd: < 5 seconden voor generatie
  • - Success rate: > 95% voor validaties
  • **Acceptabel:** Haalbaar binnen huidige architectuur
  • **Relevant:** Direct gerelateerd aan gebruikersbehoefte
  • **Tijdgebonden:** Gerealiseerd binnen huidige sprint

Criterion 1: Compliance Coverage

Gegeven AVG, BIO en justice sector vereistes

Wanneer ik de policy review

Dan zijn alle wettelijke vereisten geadresseerd

En is er een compliance matrix met wetsartikelen

En zijn boetes en sancties gedocumenteerd

En zijn er templates voor assessments

En is de approval workflow geautomatiseerd waar mogelijk

Criterion 3: Beveiliging Controls

Gegeven verschillende data classificaties

Wanneer externe data wordt gebruikt

Dan zijn de vereiste security controls gedocumenteerd

En is er een control matrix per classificatie niveau

En zijn technische implementaties gespecificeerd

En zijn compensating controls gedefinieerd

Criterion 4: Audit Trail

Gegeven gebruik van externe bronnen

Wanneer een audit plaatsvindt

Dan beschrijft de policy logging en monitoring vereisten

En zijn retention periods vastgesteld

En is er een audit log format gespecificeerd

En zijn forensic capabilities gedocumenteerd

Criterion 5: Justice Chain Integration (NIEUW)

Gegeven de justice keten samenwerking

Wanneer data wordt uitgewisseld tussen organisaties

Dan zijn chain-specifieke vereistes gedocumenteerd:

  • OM data sharing protocols
  • DJI security classifications
  • Rechtspraak openbare data regels
  • Justid identity broker policies

  • - BIO-thema 12.1: Operationele procedures
  • - BIO-thema 14.1: Beveiliging in relaties met leveranciers
  • **NEN-ISO/IEC 27001:2017 & 27002:2022**
  • - ISMS vereistes
  • - Control objectives A.5-A.18
  • - Supply chain security (A.15)
  • - Information transfer (A.13)

  • - Incident response procedures

2. Approved Sources Registry

Whitelist (Goedgekeurd)

  • Rechtspraak.nl (openbare uitspraken)
  • EUR-Lex (EU wetgeving)
  • Wetten.nl/BWB (Nederlandse wetgeving)
  • Wikipedia (algemene context)
  • Officiële overheids APIs

Greylist (Conditioneel)

  • Academic databases (met licentie)
  • News sources (met source attribution)
  • Commercial legal databases (met DPA)

Blacklist (Verboden)

  • Social media platforms
  • Ongevalideerde user content
  • Tor/dark web sources
  • Non-EU data brokers

Key Policy Elements

1. Source Approval Governance

Approval Authority Matrix

| Data Type | Approval Level | SLA |

|-----------|---------------|-----|

| Openbaar | Team Lead | 1 dag |

| Intern | Manager | 3 dagen |

| Vertrouwelijk | Director | 1 week |

| Geheim | Board | 2 weken |

2. Data Retention & Deletion

Retention Schedule

  • Definitie output: 5 jaar (wettelijk)
  • Validatie logs: 1 jaar (audit)
  • API responses: 30 dagen (cache)
  • Personal data: Minimaal noodzakelijk
  • System logs: 90 dagen (security)

Deletion Procedures

  • Automated purging via retention policies
  • Secure wipe (3x overwrite)
  • Certificate of destruction
  • Backup inclusion verificatie

3. Cross-Border Transfer Controls

EU/EEA Transfers

  • Geen additionele vereisten
  • Standard contractual clauses (SCC)

Third Country Transfers

  • Adequacy decision check
  • Transfer impact assessment (TIA)
  • Supplementary measures
  • Binding corporate rules (BCR)

4. Encryption Standards

Data at Rest

  • AES-256-GCM minimum
  • Key management via HSM
  • Database transparent encryption
  • File system encryption

Data in Transit

  • TLS 1.3 preferred, 1.2 minimum
  • Certificate pinning voor kritieke APIs
  • VPN voor management traffic
  • End-to-end encryption voor Geheim

5. Access Control Framework

RBAC Model
