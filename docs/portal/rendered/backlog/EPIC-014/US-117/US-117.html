US-117: Implementeer Validation Element Detectors

User Story

Als validatie engine

Wil ik specifieke elementen in definities kunnen detecteren

Zodat ik kan valideren of vereiste onderdelen aanwezig zijn

Status

Status: TODO

Priority: HIGH

Story Points: 5

Epic: EPIC-014: Business Logic Refactoring

Business Context

Set van 7 detectie functies die checken of bepaalde vereiste elementen aanwezig zijn in een definitie.

Acceptance Criteria

  • [ ] Implementeer unieke identificatie detector
  • [ ] Implementeer testbaar element detector
  • [ ] Implementeer onderscheidend kenmerk detector
  • [ ] Implementeer authentieke bron detector
  • [ ] Implementeer ontologische categorie detector
  • [ ] Implementeer single sentence detector
  • [ ] Implementeer noun start detector

Detection Patterns

detectors = {
    "unique_id": r"\b(uniek|specifiek|identificeer|registratie|nummer|code|id)\b",
    "testable": r"\b(\d+|binnen|na|voor|volgens|conform|gebaseerd op)\b",
    "distinguishing": r"\b(onderscheidt|specifiek|bijzonder|kenmerk|eigenschap)\b",
    "authentic": r"\b(volgens|conform|gebaseerd|bepaald|bedoeld|wet|regeling)\b",
    "ontological": r"\b(soort|type|categorie|proces|resultaat|exemplaar)\b",
    "noun_start": r"^(?!is|de|het|een|wordt|betreft|zijn|kan|moet|mag)\b"
}

Special Rules

  • Onderscheidend kenmerk: ook TRUE als definitie > 8 woorden
  • Single sentence: max 1 sentence ending (.!?)

Dependencies

  • Regex patterns
  • Text analysis utilities
