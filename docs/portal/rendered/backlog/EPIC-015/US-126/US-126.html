US-126: API Key Management

Niet nodig voor single‑user; gepland voor multi‑user. Genereren/revoken keys en usage limits.

Acceptatiecriteria (functioneel)

  • [ ] Key genereren, roteren, intrekken
  • [ ] Key‑scope en quota (basis)
  • [ ] Gebruiksmeldingen in logs (geen key in clear text)

Niet‑functioneel

  • [ ] Keys hashed/opgeslagen als secrets; niet herleidbaar in logs
  • [ ] Rate limiting per key (globaal per endpoint)

Definition of Done

  • [ ] Admin UI/CLI snippet; doc hoe keys te beheren
  • [ ] Tests voor genereren/rotatie/intrekken
