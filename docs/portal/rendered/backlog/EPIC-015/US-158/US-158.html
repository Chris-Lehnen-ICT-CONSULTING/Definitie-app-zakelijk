US-158: Authenticatie (Login/Logout)

Niet nodig voor single‑user; gepland voor multi‑user. Basis login/logout, sessiebeheer en password policy.

Probleem/Doel

  • Gebruikersidentiteiten beheren voor gedeelde omgevingen.

Acceptatiecriteria (functioneel)

  • [ ] Login met gebruikersnaam/wachtwoord; logout met sessie‑invalidering
  • [ ] Sessietimeout configurabel (idle/absolute)
  • [ ] Password policy (min‑lengte, complexiteit) afdwingbaar
  • [ ] Foutmeldingen veilig (geen user enum)

Niet‑functioneel

  • [ ] Wachtwoordsopslag met sterke hashing (bcrypt/argon2)
  • [ ] Rate limiting op login poging (basic throttling)
  • [ ] Audit event voor succes/mislukte logins

Definition of Done

  • [ ] OAS/Ondersteunende doc bijgewerkt (hoe auth werkt)
  • [ ] Basistests (happy/lockout)
  • [ ] Geen PII in logs; veilige error paths

Afhankelijkheden

  • EPIC‑015 (RBAC, API‑keys, observability)

Out‑of‑Scope

  • Federated login/SSO, MFA (latere fase)
