US-163: Validatieregels beheren (regels/gewichten/drempels)

Doel

Beheer van validatieregels (actief/inactief), prioriteit/gewichten en globale drempels voor overall_score.

Acceptatiecriteria

  • [ ] Lijst met regels (ID, naam, categorie, prioriteit/gewicht, actief ja/nee)
  • [ ] Bewerk gewichten/prioriteiten en activeer/deactiveer regels via UI
  • [ ] Opslag in DB (config) met audit; herlaadbaar zonder restart
  • [ ] Overall drempel configurabel (default 0.75)
  • [ ] Export/Import regelsconfig mogelijk

Definition of Done

  • [ ] UI‑beheer pagina
  • [ ] DB‑opslag + audit trail
  • [ ] Validatieservice leest actuele configuratie
  • [ ] Documentatie bijgewerkt

Data‑model

  • Tabel `validation_rules_config` (id, rule_id, name, category, weight, active, updated_at, updated_by)
  • Tabel `validation_global_settings` (id, key, value_json)

UI‑Details

  • Tabelweergave met filter per categorie; inline edit voor gewicht/actief.
  • Tooltip met regel‑omschrijving; link naar documentatie.

Service integratie

  • ModularValidationService leest gewichten/actief bij initialisatie of via cache refresh.

Import/Export

  • JSON/YAML mapping: lijst van regels met velden; bij import validatie en diff‑preview.

Risico’s

  • Te veel toggles kunnen gedrag onvoorspelbaar maken → audit/tracking, testprotocol bij wijzigingen.
