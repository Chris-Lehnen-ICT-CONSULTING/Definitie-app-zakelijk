US-186: Configuratie import/export (JSON/YAML)

Doel

Beheerconfiguratie kunnen exporteren (backup/overdracht) en importeren (migratie/herstel) in JSON of YAML.

Acceptatiecriteria

  • [ ] Export van huidige configuratie naar JSON/YAML via UI
  • [ ] Import met validatie en preview ('diff') voor toepassen
  • [ ] Foutmeldingen duidelijk; import atomic (alles of niets)

Definition of Done

  • [ ] UI‑acties export/import aanwezig
  • [ ] Validatie en preview implementatie
  • [ ] Documentatie bijgewerkt

Import/Export details

  • Export: download JSON/YAML met alle configgroepen (gate‑policy, regels, contextopties).
  • Import: upload → validatie → diff‑preview → confirm → atomic apply (+ audit entry).
  • Foutafhandeling: duidelijke meldingen; geen gedeeltelijke writes.

Testplan

  • Unit: export payload structureren; import validatie (schema/velden) en dry‑run
  • Integratie: diff‑preview, atomic apply; audit entry na import
  • UI: duidelijke foutmeldingen en confirmaties
