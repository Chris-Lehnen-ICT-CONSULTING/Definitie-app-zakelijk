US-006: validatie Interface Design

Gebruikersverhaal

As a software architect

wil ik a clean validatie interface design

zodat validatieregels can be added en modified easily

  • **Specifiek:** [Exact gedrag dat moet worden gerealiseerd]
  • **Meetbaar:**

  • - Processing tijd: < 5 seconden voor generatie
  • - Success rate: > 95% voor validaties
  • **Acceptabel:** Haalbaar binnen huidige architectuur
  • **Relevant:** Direct gerelateerd aan gebruikersbehoefte
  • **Tijdgebonden:** Gerealiseerd binnen huidige sprint

Criterion 1: Interface Definition

gegeven the need for multiple validatieregels

wanneer designing the validatie system

dan a clear interface is defined for all rules

Criterion 2: Rule Categories

gegeven 45 validatieregels in 7 categories

wanneer organizing the validatie system

dan rules are properly categorized (ARAI, CON, ESS, INT, SAM, STR, VER)

Criterion 3: Extensibility

gegeven new validatieregels may be needed

wanneer the interface is designed

dan new rules can be added without changing existing code

Technische Notities

  • Interface defined in `src/services/validatie/interfaces.py`
  • Rule categories match ASTRA richtlijnen
  • Each rule implements `ValidationRule` interface
  • Supports priority levels (HOOG, GEMIDDELD, LAAG)
