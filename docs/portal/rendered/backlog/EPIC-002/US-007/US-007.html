Gebruikersverhaal

As a developer

wil ik the core validatie engine geïmplementeerd

zodat all 45 validatieregels can be executed

Acceptatiecriteria

SMART Acceptatiecriteria

  • **Specifiek:** [Exact gedrag dat moet worden gerealiseerd]
  • **Meetbaar:**
  • - Response tijd: < 200ms voor UI acties
  • - Processing tijd: < 5 seconden voor generatie
  • - Success rate: > 95% voor validaties
  • **Acceptabel:** Haalbaar binnen huidige architectuur
  • **Relevant:** Direct gerelateerd aan gebruikersbehoefte
  • **Tijdgebonden:** Gerealiseerd binnen huidige sprint

Criterion 1: Rule Execution

gegeven a definition to validatie

wanneer validatie is triggered

dan all applicable rules are executed

Criterion 2: Result Collection

gegeven multiple validatieregels run

wanneer rules complete

dan results are collected en aggregated

Criterion 3: Error Handling

gegeven a validatieregel fails

wanneer the error occurs

dan it doesn't stop other rules from running

Technische Notities

  • Core engine in `src/services/validatie/modular_validation_service.py`
  • Implements all 45 validatieregels
  • Dual JSON+Python format for rules
  • Rules organized in `src/toetsregels/regels/` En `config/toetsregels/regels/`
