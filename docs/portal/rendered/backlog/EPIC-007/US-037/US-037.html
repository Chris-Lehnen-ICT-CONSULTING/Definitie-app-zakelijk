US-037: Database indexering & query‑tuning (begrip/status/categorie; N+1 elimineren)

Gebruikersverhaal

Als data engineer wil ik correcte indexen en efficiënte queries zodat zoekopdrachten en overzichten snel blijven bij groei.

Acceptatiecriteria (SMART)

  • Specifiek: indexen op `begrip`, `status`, `categorie`; refactor zoekpaden; elimineer N+1 patronen.
  • Meetbaar: search p95 < 50ms op 10k records (synthetische dataset); geen full scans in explain waar onnodig.
  • Tijdgebonden: sprint‑38.

Testplan

  • Benchmark tests met pytest‑benchmark en testdatabase; explain‑analyse screenshots/logs.

Definition of Done

  • Migraties aanwezig; README “Database Richtlijnen” aangevuld; benchmarks opgenomen in reports.
