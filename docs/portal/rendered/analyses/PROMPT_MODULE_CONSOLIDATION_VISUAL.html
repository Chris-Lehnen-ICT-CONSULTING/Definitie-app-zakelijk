Contradiction Resolution

Issue 1: Kick-off Terms (CRITICAL)

Before:

# SemanticCategorisationModule (line 197-206)
KICK-OFF opties:
- 'activiteit waarbij...'  ✅ INSTRUCTED
- 'handeling die...'        ✅ INSTRUCTED
- 'proces waarin...'        ✅ INSTRUCTED

# ErrorPreventionModule (line 178-179)
forbidden_starters = [
    "proces waarbij",       ❌ FORBIDDEN
    "handeling die",        ❌ FORBIDDEN
    ...
]

RESULT: Direct contradiction! AI receives conflicting instructions.

After:

# CategoryGuidanceModule (merged, clarified)
KICK-OFF opties (ZELFSTANDIGE NAAMWOORDEN):
- 'activiteit waarbij...'  ✅ NOUN (handelingsnaamwoord)
- 'handeling die...'        ✅ NOUN (handelingsnaamwoord)
- 'proces waarin...'        ✅ NOUN (handelingsnaamwoord)

# ValidationRulesModule (forbidden patterns section)
VERBODEN KOPPELWERKWOORDEN:
- "is een proces"           ❌ VERB (koppelwerkwoord)
- "betekent een handeling"  ❌ VERB (koppelwerkwoord)
- "omvat activiteiten"      ❌ VERB (werkwoord)

TOEGESTAAN (zelfstandige naamwoorden):
- "proces dat..."           ✅ NOUN
- "activiteit waarbij..."   ✅ NOUN
- "handeling die..."        ✅ NOUN

RESOLUTION: Clarify that "proces", "activiteit", "handeling" are NOUNS
            (handelingsnaamwoorden), not verbs. The forbidden pattern
            is starting with VERBS ("is", "betekent"), not NOUNS.

Before:

# SemanticCategorisationModule
ontological_category = "proces" | "type" | "resultaat" | "exemplaar"
context.set_shared("ontological_category", ...)

# TemplateModule (BROKEN!)
category = context.get_metadata("semantic_category")  # NEVER EXISTS!
# Expects: "Proces" | "Object" | "Actor" | ...

# ErrorPreventionModule (legacy)

RESULT: 3 incompatible naming schemes, TemplateModule never runs.

After:

# CategoryGuidanceModule (unified)
ontological_category = "proces" | "type" | "resultaat" | "exemplaar"
context.set_shared("ontological_category", category)

# All modules use consistent naming:
- PRIMARY: ontological_category (lowercase)
- VALUES: proces, type, resultaat, exemplaar
- DEPRECATED: semantic_category, domain_contexts

RESOLUTION: Single source of truth for category naming.

Issue 3: Hardcoded Rules (HIGH)

Before:

# StructureRulesModule (line 132-332)
def _build_str01_rule(self): ...
def _build_str02_rule(self): ...
def _build_str03_rule(self): ...
# ... 9 hardcoded methods

# IntegrityRulesModule (similar pattern)
# ... hardcoded methods

PROBLEM: Bypasses cached_toetsregel_manager
         Defeats US-202 optimization (77% speedup)
         Rules may drift from JSON definitions

After:

# ValidationRulesModule (unified)
def execute(self, context):
    manager = get_cached_toetsregel_manager()
    all_rules = manager.get_all_regels()

    for category in ["ARAI", "CON", "ESS", "STR", "INT", "SAM", "VER"]:
        category_rules = {k: v for k, v in all_rules.items()
                          if k.startswith(f"{category}-")}
        sections.append(self._format_category(category, category_rules))

RESOLUTION: All rules loaded from cache (single source of truth)
            Consistent with other rule modules
            US-202 compliant (no duplicate loading)
